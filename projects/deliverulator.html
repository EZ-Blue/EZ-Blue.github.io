Deliverulator: A Campus Delivery Route Optimization Simulation

Introduction

The last-mile delivery challenge has become increasingly critical as e-commerce and on-demand services continue to expand. With the global logistics market projected to reach $12.68 trillion by 2023, optimizing delivery routes and resource allocation has significant implications for economic efficiency, environmental sustainability, and customer satisfaction. Particularly in urban environments like university campuses, where delivery tasks are frequent but infrastructure is often constrained, efficient routing algorithms can dramatically reduce delivery times, energy consumption, and operational costs.

Deliverulator addresses these challenges through an interactive simulation that demonstrates the practical application of pathfinding algorithms in a real-world context. Developed as a final project for the Engineering Computation course at Carnegie Mellon University, this C++ application simulates a delivery management system where users assign delivery robots to orders across campus locations, allowing for experimentation with different assignment strategies and visualizing their effects on efficiency.

The significance of this work extends beyond academic exercise; it provides insights into how computational methods can optimize resource allocation in last-mile delivery scenarios. By visualizing the pathfinding process and allowing users to experiment with different assignment strategies, Deliverulator bridges the gap between abstract algorithms and practical applications in logistics management.

Methods

System Architecture

Deliverulator is built as a modular C++ application with four main components:

  1. User Interface (UI) Module: Handles window management, user input, order generation, and visualization of the simulation state. Implemented in the ui.cpp file, this module creates an interactive environment where users can select robots, assign orders, and observe the delivery process in real-time.
  2. Robot Control System: Manages the state and behavior of delivery robots, including movement along computed paths and order fulfillment logic. Each robot has positioning data, movement capabilities, and order tracking functionality as shown in the robot.cpp implementation.
  3. Path Planning Algorithm: Implements Dijkstra's algorithm to compute optimal routes between locations on the campus map. The path-planner.cpp file contains the core pathfinding logic that calculates the shortest path from a robot's current position to an order's pickup location, and then to the delivery destination.
  4. Map System: Represents the navigable environment as a graph where nodes correspond to buildings and other landmarks on campus, connected by edges representing feasible paths. This system provides the topographical data necessary for pathfinding.

Pathfinding Implementation

The route optimization in Deliverulator is powered by Dijkstra's algorithm, which computes the shortest path between any two points on the campus map. The implementation consists of:

  1. Graph Representation: The campus map is modeled as a weighted graph where nodes represent locations and edges represent paths between them, with weights corresponding to distances.
  2. Dijkstra's Algorithm: The core pathfinding function processes the graph using a priority queue to efficiently identify the shortest path between locations:

std::vector<const Node*> dijkstra(const Node& startNode, const Node& endNode){

    // Initialize data structures

    std::unordered_map<const Node*,PathInfo> traveledNodes;

    std::priority_queue<NodeFloatPair, std::vector<NodeFloatPair>,NodeFloatPairComparator> pq;

   

    // Add starting point

    pq.push(NodeFloatPair(&startNode,0));

   

    // Process nodes in order of increasing distance

    while(!pq.empty()){

        // Extract next closest node

        curNode = pq.top();

        pq.pop();

       

        // Check if we've processed this node before

        if (traveledNodes[curNode.node].visited == true){

            continue;

        }

       

        // Mark as visited and update distance

        traveledNodes[curNode.node].visited = true;

       

        // Process neighbors

        for (auto neighbor : curNode.node->neighbors){

            // Calculate distance to this neighbor

            curDist = getEdgeCost(*curNode.node,*neighbor) + traveledNodes[curNode.node].distance; 

           

            // Update if shorter path found

            if (curDist < traveledNodes[neighbor].distance){

                traveledNodes[neighbor].distance = curDist;

                traveledNodes[neighbor].prev = curNode.node;

                pq.push(NodeFloatPair(neighbor,curDist));

            }

        }

       

        // Stop if we've reached the destination

        if (curNode.node == &endNode){

            break;

        }

    }

    return path;

}

Results

Deliverulator successfully demonstrates the application of algorithmic pathfinding to optimize delivery routes in a campus environment. The simulation's results can be evaluated along several dimensions:

Pathfinding Efficiency

The implementation of Dijkstra's algorithm consistently generates optimal paths between campus locations. Performance metrics include:

  1. Path Optimality: Routes always follow the shortest possible path between locations, minimizing travel distance and time.
  2. Computation Speed: Path calculations are performed in real-time with negligible delay, even when multiple robots are active simultaneously.
  3. Handling of Complex Networks: The algorithm successfully navigates the complex network of paths across the CMU campus map, accounting for all available routes between buildings.

User Experience Testing

Feedback from user testing indicates that the interface effectively enables users to:

  1. Make Informed Decisions: The visualization of robots, orders, and campus geography allows users to strategically assign deliveries.
  2. Observe Algorithm Effects: Users can clearly observe how different assignment strategies impact overall delivery efficiency.
  3. Understand Resource Allocation: The simulation demonstrates the effects of robot availability and positioning on delivery capabilities.

Educational Value

As an educational tool, Deliverulator illustrates several important computational concepts:

  1. Graph Theory Application: Demonstrates the practical application of graph representation and traversal.
  2. Algorithm Visualization: Makes abstract algorithms tangible by visualizing their execution in a familiar context.
  3. Resource Optimization: Shows how computational methods can optimize resource allocation in real-world scenarios.

Discussion

Practical Applications and Social Value

Deliverulator, while developed as an educational simulation, addresses challenges relevant to real-world logistics operations:

  1. Campus Delivery Optimization: Universities and corporate campuses could apply similar algorithms to streamline internal mail and package delivery services, reducing operational costs and improving service times.
  2. Resource Allocation Training: The interactive nature of the simulation makes it an effective training tool for logistics personnel learning to make efficient routing decisions.
  3. Energy and Emissions Reduction: By optimizing routes, delivery systems based on similar algorithms could reduce fuel consumption and emissions—especially important as delivery services continue to expand globally.
  4. Educational Impact: As a teaching tool, the project bridges theoretical computer science concepts with practical applications, helping students understand the real-world impact of algorithms.

Limitations and Future Work

While Deliverulator successfully demonstrates core concepts in route optimization, several limitations and opportunities for enhancement exist:

  1. Dynamic Obstacles: The current implementation assumes static paths without temporary obstacles or closures. A more realistic model would account for path availability changes over time.
  2. Multi-Objective Optimization: Future versions could incorporate additional factors beyond distance, such as delivery priority, vehicle capabilities, or energy consumption.
  3. Machine Learning Integration: Adding predictive capabilities could allow the system to anticipate order patterns and preposition robots accordingly.
  4. Expanded Visualization: Enhanced visualization of metrics like total distance traveled, delivery times, and efficiency comparisons would provide deeper insights into optimization effects.

Conclusion

Deliverulator demonstrates the successful application of pathfinding algorithms to the last-mile delivery challenge in a campus environment. By providing an interactive visualization of how computational methods can optimize routing decisions, the project bridges the gap between abstract algorithms and practical logistics applications.

The project's educational value extends beyond the classroom, offering insights into how similar computational approaches can address real-world challenges in resource allocation, energy efficiency, and service optimization. As delivery services continue to expand globally, the principles demonstrated in Deliverulator have increasing relevance to economic, environmental, and social considerations in urban logistics.

Through its combination of algorithmic implementation, visual representation, and interactive decision-making, Deliverulator serves as both an educational tool and a prototype for how computational thinking can be applied to optimize systems that impact daily life.
